# M.A.S.T.E.R. System - 技术规格说明书

## 1. 项目概述

**项目名称**：M.A.S.T.E.R. (Multi-Agent Synergized Task Execution & Result Integration System)

**核心愿景**：构建一个基于 Web 端的去中心化多 AI 协同平台，通过决策 AI 的智能调度，整合多个垂直领域 AI 的专业能力，实现复杂任务的高效拆解、并行处理与成果融合。

**目标用户**：需要复杂任务处理能力的技术爱好者、小团队、开发者。

## 3. 核心功能模块

### 3.1 决策 AI (Decision Agent)

**职责**：
- 解析用户输入，识别任务意图
- 根据任务类型拆分子任务
- 为每个子任务匹配最合适的 AI 节点
- 汇总结果，做最终整合决策

**能力画像系统**：

**任务分配算法**：
1. 解析任务关键词，生成需求向量
2. 计算每个 AI 节点与需求的余弦相似度
3. 选取相似度最高的 Top-K 节点
4. 根据任务复杂度决定并行或串行执行

### 3.2 协作模式

#### 标准分工模式 (Standard Division)
- 决策 AI 拆解任务 → 分发给指定 AI → 接收结果 → 整合输出

#### 讨论共识模式 (Consensus Discussion)
- 广播任务给所有相关 AI
- 循环收集各 AI 观点
- 达到共识或迭代阈值后终止

#### 评审团模式 (Jury Evaluation)
- 多个 AI 独立完成任务
- 互评打分（多维度）
- 决策 AI 计算评分，决定是否采纳或回退

### 3.3 通信协议

采用 JSON-RPC 风格的内部协议：

```json
{
  "protocol": "MASTER v1.0",
  "type": "task_dispatch | result_return | consensus_request | evaluation",
  "sender": "decision_agent",
  "receiver": "writer",
  "payload": {
    "task_id": "uuid",
    "sub_task": "具体任务描述",
    "context": {},
    "mode": "standard | consensus | jury"
  }
}
```

## 4. API 设计

### 4.1 后端 API

| 方法 | 路径 | 说明 |
|:---|:---|:---|
| POST | `/api/tasks` | 创建新任务 |
| GET | `/api/tasks/{task_id}` | 获取任务状态和结果 |
| GET | `/api/agents` | 获取所有 AI 节点状态 |
| POST | `/api/agents/{agent_id}/test` | 测试单个 AI 连接 |
| GET | `/api/history` | 获取历史任务列表 |

### 4.2 WebSocket

| 事件 | 说明 |
|:---|:---|
| `task_progress` | 任务进度实时推送 |
| `agent_status` | AI 节点状态变更 |
| `consensus_round` | 讨论共识模式每一轮的结果 |
